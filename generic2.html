Data Normalisation

In unit 6, we discussed database design and normalisation. According to Conolly and Begg, (2014) database design is the process of developing a formal plan for how a database will store and manage data. It entails establishing the database structure, tables, relationships, and restrictions to provide effective data storage, retrieval, and maintenance. Some crucial aspects consider during the database design process includes requirement analysis, creation of an entity relation diagram, normalisation etc.

Normalisation is the process of reorganising data within a database so that users can use it for additional queries and analysis (Simplilearn, 2021). In a nutshell, it is the procedure for producing clean data. This includes removing superfluous and unstructured data and making all entries and fields appear comparable. 

According to Datanamic Solutions (2019), the most common normalization forms are:

  1. First Normal Form (1NF): Each table should have a primary key, and all attributes must be atomic, meaning they cannot be further divided. Therefore, an entity in its first normal form has no repeated groups and all attributes must have unique names.
  2. Second Normal Form (2NF): A table should be in 1NF, and all non-key attributes should be fully functionally dependent on the entire primary key. This means an entity with a primary key consisting of only one attribute is in the second normal form (2NF).
  3. Third Normal Form (3NF): A table should be in 2NF, and all attributes should be functionally dependent on the primary key (no transitive dependencies).
